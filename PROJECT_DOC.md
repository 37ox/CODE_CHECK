# C/C++ 风险扫描项目文档

## 1. 方案、完成内容、目标达成

### 1.1 采用的方案

- **总体方案**：启发式静态扫描（非完整 AST 编译器前端），以“函数实现”为最小分析单元。
- **扫描粒度**：函数级独立扫描。每个函数单独做规则检查，不强依赖跨文件调用关系。
- **输入范围**：递归收集 `.h/.hpp/.hh/.hxx/.cpp/.cc/.cxx`。
- **函数筛选模式**：
  - 默认：扫描输入路径中的全部函数定义（`all_definitions`）。
  - `--declared-only`：仅扫描头文件声明名匹配到的函数定义。
- **规则引擎方向**：围绕 3 类风险构建通用规则：
  - 空指针访问风险
  - 数组越界访问风险
  - 除 0 / 近零除数风险
- **输出形态**：
  - 中文文本报告（按三大类分组+计数+明细）
  - JSON 报告（含中英文风险字段、汇总字段）

### 1.2 已完成内容

1. **扫描基础能力**
   - 文件收集、文本读取、注释屏蔽（减少注释干扰）。
   - 函数定义提取与函数参数解析。
2. **空指针规则**
   - 识别 `*p`、`p->x`、`p[i]` 解引用点。
   - 识别 `if (p)`、`if (p != nullptr)`、`if (!p) return`、`assert(p)` 等保护模式。
   - 支持分支作用域与“重新赋值失效（epoch）”机制：
     - 指针被重新赋值（尤其 `NULL/nullptr/0`）后，之前保护失效，后续必须重新保护。
3. **数组越界规则**
   - 识别 `arr[idx]` / `vec[i]` 访问点。
   - 同时要求**上界**与**下界**保护（例如 `i < size` 且 `i >= 0`）。
   - 支持早退出拒绝式保护（如 `if (i < 0 || i >= size) return;`）。
   - 索引表达式做整数性判定，非明显整型索引会直接报风险。
4. **除 0 与近零除数规则**
   - 识别 `/`、`%` 分母表达式。
   - 检测显式 0 分母与缺失非零保护。
   - 增加近零保护规则（阈值 `1e-20`），支持 `abs/fabs` 风格判断。
   - 对 `if` 条件做布尔表达式解析（`&& / || / !`）+ 区间推断，减少分支误报（例如 `ttt9`）。
5. **结果展示**
   - 文本输出按你指定模板分三段：空指针、越界、除0。
   - JSON 输出含 `summary` 与 `summary_zh`，可被后续工具链消费。

### 1.3 达成的目标

- 达成“**函数级独立扫描**”目标，避免把某个文件的检测强绑定另一个文件声明。
- 达成“**三类核心风险统一规则化检查**”目标，避免按单用例硬编码 if/else 覆盖。
- 达成“**近零除数风险**”目标，不仅检查 `== 0`，也检查极小值导致的数值风险。
- 达成“**中文报告** + **结构化 JSON**”双输出目标，便于人读和系统集成。
- 当前自测（`python cpp_risk_scanner.py ./sample`）输出：
  - 空指针访问风险：6 处
  - 数组越界访问风险：6 处
  - 除0风险（含近零）：4 处
  - 其中 `ttt9` 当前不再误报。

## 2. 当前可覆盖场景（细化）

### 2.1 空指针访问风险

**可覆盖场景**

- 函数参数是指针，出现 `*p` / `p->m` / `p[i]` 解引用。
- 解引用前存在显式保护：
  - `if (p) { ... }`
  - `if (p != nullptr) { ... }`
  - `if (p == nullptr) return;`
  - `if (!p) return;`
  - `assert(p)` / `CHECK(p)` / `CHECK_NOTNULL(p)`
- 保护作用域识别：
  - `if` 分支内受保护，分支外不会错误继承保护。
- 指针重置识别：
  - 保护后若 `p = NULL/nullptr/0` 或其他赋值，旧保护失效，后续解引用会重新评估。

**典型能识别的问题**

- 仅在分支里判空，分支外继续用指针。
- 判空后中途重置为 `nullptr`，后续再次解引用。
- 解引用发生在没有任何保护的位置。

### 2.2 数组越界访问风险

**可覆盖场景**

- 下标访问点：`container[index]`。
- 索引表达式整数性检查：
  - 纯整数字面量、整型变量、整型强转表达式可通过。
  - 可疑非整型索引会报风险。
- 上下界保护检查：
  - 上界：`i < vec.size()`、`vec.size() > i` 等。
  - 下界：`i >= 0`、`0 <= i`、`for (int i = 0; ...)` 等。
  - 无符号索引类型可作为下界保护线索。
- 早退出保护识别：
  - `if (i < 0 || i >= vec.size()) return;` / `throw` / `break` / `continue`。

**典型能识别的问题**

- 只校验上界，不校验负索引。
- 下标未做边界判断直接访问。
- 使用可疑非整型表达式做下标。

### 2.3 除 0 风险

**可覆盖场景**

- 除数是常量 0（直接命中）。
- 除数是变量但无非零保护。
- 可识别保护：
  - `b != 0`
  - `if (b == 0) return;`（含早退出）
  - `assert(b != 0)` / `CHECK(b != 0)`
  - `if (b) ...`
- 支持条件推断：
  - 解析 `&& / || / !` 与比较表达式，推断当前分支下是否可证明 `b != 0`。

**典型能识别的问题**

- 只做了部分条件判断，无法证明分母非零。
- 分支外使用除法但非零保护只在分支内成立。

### 2.4 近零除数风险（浮点数值安全）

**可覆盖场景**

- 浮点分母（参数或局部浮点变量）即使 `!= 0`，仍检查是否缺少“远离 0”保护。
- 阈值规则：`|den| < 1e-20` 视为近零风险域。
- 可识别保护：
  - `abs(den) > eps` / `fabs(den) > eps`（`eps >= 1e-20`）
  - `if (abs(den) <= eps) return;` 这类拒绝式保护
  - 区间条件组合可推断分支已远离 0

**典型能识别的问题**

- 只写 `den != 0`，没有 `abs(den) > eps` 类保护。
- 条件逻辑复杂但仍可能落入 `(-1e-20, 1e-20)` 风险区间。

### 2.5 结果与集成场景

**当前可直接支持的使用方式**

- 命令行批量扫描目录或文件。
- 只扫声明接口（`--declared-only`）或扫全部定义（默认）。
- 输出中文文本给研发自查；输出 JSON 给 CI/脚本二次处理。

**当前边界（已知）**

- 启发式规则，不是完整语义分析，仍可能存在误报/漏报。
- 宏展开、模板深层实例化、跨函数数据流/跨文件别名传播不做完整求解。
- 建议流程仍是“自动扫描 + 人工复核”。
